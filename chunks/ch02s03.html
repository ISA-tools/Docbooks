Using BII Data Manager

Data Loader Function

BII-Data-Manager is part of the ISA infrastructure. As such, it shares a number of behavior with other tools such as ISAcreator. In particular, the first thing to do in order to run the tool is to select an ISAconfiguration:

Once done, BII-Data-Manager screen will be displayed:

To load, click the top left button:

  1. Select a study archive for loading

  2. Click on load. BII-Data-Manager first validates the submission

    then loads

    By default, at the end of a successful loading, BII-Data_manager will prompt the security management screens.

    The function allows to grant permissions to access the newly loaded dataset by select among the users as well decided the keep the datasets private or public.

Data Unloader Function

In order to remove datasets from the BII database, go back to main menu of the BII-Data-Manager

and hit the top middle button:

BII-Data-Manager will then query the database and return a list of its content so users can select which study or studies should be unload. One or more studies may be selected at any one time.

Note

Known issues: During the first selection, the button not operate. Click away, for example by trying to select another study and then click back to select the study again. This issue will be addressed in the coming release and is currently tracked under feature request #68 under github/BII-data-manager.

Indexing Function

Indices may become corrupted. It is therefore necessary to be able to regenerate the lucene index to ensure accurate searches and display of the database content

BII-Data-Manager provides such functionality, available by clicking the following button:

Simply hit the button to trigger reindexing of the BII database content.

Index creation failure will be notified by the following icon.

If index creation fails, trying running the process again. If this fails again, try unloading the study last loaded and reindex.
