Using ISAcreator

Following successfull login, users will be presented with ISAcreator Main Menu

The menu items are self explanatory.

  1. The first item allows to create a new study report from scratch

  2. The second item allows to view an existing report either for addition and simply for browsing

  3. The third item allows users to groups studies into investigations

However, before going in details in those key elements, we will focus on the next 2 menu items.

The reason for this?

Those 2 choices enable users to further customize ISAcreator installations. So let's review what those items can do for you by answering the following 2 questions!

Why does ISAcreator require an ISA configuration to run?

Different communities have different needs. So ISAconfigurations should be considered as 'skins' for ISAcreator. ISAconfigurations allow users to set annotation requirements, determine which terminologies and which controled vocabularies should be used, provide contextual help about the various fields, decide if a field is required or not and can accept a data file or not. If you are working on Drosophila, you probably don't need to see Mouse anatomy terms when looking for resources. ISAconfigurations allows to just do that. For more information about how to create configurations, please refer to ISAconfigurator user manual

Therefore, ISAcreator allows you to swap configurations depending on the study you need to report. And the good thing is that you don't need to restart the application. Just save the current study you may be working on, go to the main menu and select the , click on

Menu Item

The action will bring the configuration selection screen
